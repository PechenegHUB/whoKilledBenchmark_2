# «Скажи мне, кто твой шлиф» — запуск проекта

Streamlit-приложение для анализа панорамных микрофотографий полированных шлифов. Проект объединяет CNN-классификацию, OpenCV-метрики, superpixel-сегментацию, карты уверенности, Grad-CAM/talc-score и экспертную доразметку масок.

---

## 1. Структура проекта

После распаковки проекта в корне должны лежать основные файлы:

```text
project/
├── app.py                         # главная точка входа Streamlit-приложения
├── requirements.txt               # Python-зависимости
├── run_app.bat                    # запуск на Windows
├── run_app.sh                     # запуск на Linux/macOS
├── models/                        # веса моделей и joblib-пайплайны
├── src/                           # логика инференса, визуализации, отчётов и сохранения
├── components/                    # браузерный редактор масок
├── .streamlit/config.toml         # настройки Streamlit
└── outputs/                       # результаты работы приложения
```

Основные модули в `src/`:

```text
src/
├── inference.py                   # общий пайплайн анализа изображения
├── ml_adapter.py                  # загрузка моделей и инференс CNN/superpixel
├── classification.py              # CV-метрики, маски, классификация
├── visualization.py               # overlay, карты, цветовые маски
├── reporting.py                   # PDF-отчёты
├── review_store.py                # журнал, JSON/CSV, active learning
├── image_io.py                    # чтение и сохранение изображений
├── gis_export.py                  # GeoJSON/GIS-экспорт
└── frontend_mask_editor.py        # подключение браузерного редактора масок
```

## 2. Требования

Рекомендуемая версия Python:

```text
Python 3.10 или 3.11
```

Почему лучше не брать случайную версию Python: проект использует `torch`, `torchvision`, `segmentation-models-pytorch`, `opencv`, `scikit-image` и `streamlit`, поэтому стабильнее запускать его в отдельном виртуальном окружении.

Проверьте версию Python:

```bash
python --version
```

или:

```bash
python3 --version
```

## 3. Установка на Windows

Перейдите в папку проекта:

```bat
cd путь\к\папке\project
```

Создайте виртуальное окружение:

```bat
python -m venv .venv
```

Активируйте его:

```bat
.venv\Scripts\activate
```

Обновите `pip`:

```bat
python -m pip install --upgrade pip
```

Установите зависимости:

```bat
pip install -r requirements.txt
```

Запустите приложение:

```bat
streamlit run app.py
```

или через готовый файл:

```bat
run_app.bat
```

После запуска Streamlit обычно откроет страницу автоматически. Если браузер не открылся, перейдите вручную:

```text
http://localhost:8501
```

## 5. Модели и папка `models/`

По умолчанию приложение ищет модели в папке:

```text
models/
```

В боковой панели можно указать либо путь к папке `models`, либо путь к конкретному файлу весов.

Поддерживаемые файлы:

```text
models/
├── best_intergrowth_3class_b0_256.pt     # CNN EfficientNet-B0 для 3 классов руды
├── ensemble_pipeline.joblib              # late fusion / meta-classifier
├── model.joblib                          # superpixel-модель, если она сохранена под таким именем
├── superpixel_model.joblib               # альтернативное имя superpixel-модели
├── segmentation_superpixel.joblib        # альтернативное имя superpixel-модели
└── best_model.pth                        # fallback-имя для старой torch/SMP-сегментации
```

Если в `models/` лежит `best_intergrowth_3class_b0_256.pt`, приложение пытается загрузить CNN-классификатор EfficientNet-B0. Если дополнительно найден `ensemble_pipeline.joblib`, подключается ансамбль late fusion. Если найден `model.joblib` или другой superpixel-joblib, подключается superpixel-сегментация.

Если веса не найдены или не загрузились, приложение не падает: оно запускается в `cv_baseline`-режиме. Такой режим подходит для демонстрации интерфейса, но качество будет ниже, потому что нейросетевая часть не используется.

## 6. Запуск анализа одного изображения

1. Откройте приложение.
2. В боковой панели проверьте путь к `models/`.
3. Выберите устройство:
   - `auto` — приложение само выберет CUDA, если она доступна;
   - `cuda` — принудительно GPU;
   - `cpu` — принудительно процессор.
4. Установите порог оталькованности в процентах.
5. Перейдите во вкладку **«Анализ одного шлифа»**.
6. В блоке **«Что считать в этом запуске»** выберите нужные расчёты:
   - базовые метрики, итоговый класс и итоговая маска;
   - superpixel-сегментация;
   - подробная карта `talc-score`;
   - Grad-CAM;
   - карта общей уверенности;
   - сравнение сегментаций;
   - overlay-картинки для ZIP/PDF.
7. Загрузите изображение в формате `PNG`, `JPG`, `JPEG`, `TIF` или `TIFF`.
8. Нажмите **«Запустить анализ»**.

После завершения приложение покажет исходник, итоговую маску, класс образца, долю талька, метрики, карты уверенности и доступные визуализации.

## 7. Пакетная обработка

Пакетный режим нужен, когда нужно обработать сразу несколько шлифов.

1. Перейдите во вкладку **«Пакетная обработка»**.
2. Заполните метаданные партии:
   - название партии;
   - месторождение;
   - масштаб;
   - микроскоп;
   - оператор;
   - комментарий.
3. Загрузите несколько изображений или ZIP-архив с изображениями.
4. В блоке **«Что считать в этом запуске»** выберите, какие артефакты нужны для партии.
5. Проверьте предпросмотр файлов.
6. Нажмите **«Запустить пакетный анализ»**.

После обработки можно:
- отфильтровать результаты по партии;
- посмотреть таблицу по всем изображениям;
- открыть отдельный образец из партии;
- выбрать тип маски для просмотра;
- выбрать карту уверенности;
- сформировать PDF-отчёт;
- собрать ZIP с outputs;
- собрать GIS/GeoJSON-экспорт.

## 8. Экспертная проверка и ручная правка маски

Во вкладке **«Экспертная проверка и доразметка»** можно открыть сохранённый результат и проверить предсказание модели.

Доступные действия:

- посмотреть исходное изображение;
- посмотреть маску модели;
- посмотреть overlay;
- изменить итоговый класс;
- добавить комментарий эксперта;
- вручную исправить маску в браузерном редакторе;
- использовать масштабирование;
- использовать ластик;
- сохранить экспертную оценку;
- сохранить экспертную маску и пару для active learning.

Экспертная правка нужна, чтобы спорные зоны не терялись: исправленные маски можно потом использовать для дообучения модели.

Во вкладке **«Журнал и экспорт»** доступны все сохранённые результаты.

## 11. Быстрый режим и полный режим

Если нужно быстро проверить партию, можно выключить тяжёлые артефакты:

- `Superpixel-сегментация`;
- `Подробная карта talc-score`;
- `Grad-CAM`;
- `Карта общей уверенности`;
- `Сравнение сегментаций`;
- `Overlay-картинки для ZIP/PDF`.

Базовые метрики, итоговый класс и итоговая маска всегда считаются. Их нельзя отключить, потому что без них невозможны доля талька, классификация и экспертная проверка.

Рекомендуемые режимы:

```text
Быстрое демо:
- оставить только базовые метрики;
- выключить Grad-CAM, talc-score, superpixel и overlay.

Экспертный разбор:
- включить talc-score;
- включить карту уверенности;
- включить Grad-CAM;
- включить сравнение сегментаций.

Финальный прогон партии:
- включить superpixel;
- включить overlay;
- включить PDF/ZIP-выгрузку после обработки.
```

## 12. Частые проблемы

### `ModuleNotFoundError`

Значит зависимости не установлены или активировано не то окружение.

Решение:

```bash
pip install -r requirements.txt
```

Проверьте, что окружение активно:

```bash
where python
```

на Windows или:

```bash
which python
```

на Linux/macOS.

---

### Streamlit не запускается

Попробуйте запуск через модуль Python:

```bash
python -m streamlit run app.py
```

Если порт занят:

```bash
streamlit run app.py --server.port 8502
```

---

### CUDA не работает

Поставьте в боковой панели устройство `cpu`.

Если нужен GPU, проверьте:

```bash
python -c "import torch; print(torch.cuda.is_available())"
```

Если вывод `False`, значит PyTorch не видит CUDA.

---

### Модель не загружается

Проверьте:

1. существует ли папка `models/`;
2. лежит ли в ней нужный `.pt`, `.pth` или `.joblib`;
3. совпадает ли архитектура весов с ожидаемой;
4. нет ли кириллицы или пробелов в пути, если запуск идёт на Windows.

Если модель не загрузилась, приложение перейдёт в `cv_baseline`-режим и покажет сообщение в блоке **«Статус модели»**.

---

### Большие TIFF долго обрабатываются

Это нормально для панорамных изображений. Для ускорения:

- выключите Grad-CAM;
- выключите talc-score;
- выключите superpixel-сегментацию;
- выключите overlay для ZIP/PDF;
- запускайте пакетный анализ на GPU, если доступно;
- используйте меньшее количество изображений в одном ZIP.

---

### После изменения файлов интерфейс показывает старое состояние

Очистите кэш Streamlit:

```bash
streamlit cache clear
```

После этого перезапустите приложение:

```bash
streamlit run app.py
```
