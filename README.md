# Норникель — Human-in-the-loop обёртка

Streamlit-приложение для задачи «Скажи мне, кто твой шлиф».

## Что умеет

1. Загружает OM-изображение шлифа PNG/JPG/TIFF.
2. Сегментирует зоны скопления талька.
3. Считает долю зон талька от площади изображения.
4. Классифицирует руду.
5. Поддерживает пакетную обработку нескольких изображений.
6. Формирует очередь экспертной проверки.
7. Даёт эксперту подтвердить или изменить класс.
8. Даёт эксперту вручную исправить маску через canvas:
   - голубая кисть — добавить тальк;
   - красная кисть — удалить лишнюю маску.
9. Сохраняет экспертную маску, комментарий, финальный класс, JSON/CSV и active-learning пары.

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
streamlit run app.py
```

или:

```bash
./run_app.sh
```

На Windows:

```bat
run_app.bat
```

## Модель

По умолчанию приложение ищет веса здесь:

```text
models/best_model.pth
```

Если весов нет, приложение запускается в `cv_baseline`-режиме, чтобы можно было показать интерфейс и экспертную доразметку.

Ожидаемая архитектура для весов из старого ноутбука:

```python
segmentation_models_pytorch.Unet(
    encoder_name="efficientnet-b5",
    encoder_weights=None,
    in_channels=3,
    classes=4,
)
```

Тальк берётся как класс `3` для 4-классовой модели.

## Outputs

```text
outputs/
├── images/           # входные изображения
├── masks/            # маски модели
├── expert_masks/     # исправленные экспертные маски
├── overlays/         # overlay и diff-карты
├── reviews/          # CSV + JSON экспертных решений
└── active_learning/  # image + expert_mask для дообучения
```

## Как демонстрировать жюри

Формулировка:

> Система не заменяет геолога, а ускоряет его работу. Модель автоматически находит зоны скопления талька, считает долю талька и предлагает класс руды. Эксперт может подтвердить результат или исправить маску вручную. Все исправления сохраняются как новая обучающая выборка для active learning.

## Исправления в этой версии

- Исправлена ручная разметка: цвет кисти «Добавить тальк» теперь совпадает с `TALC_BLUE_HEX = #0060FF`, поэтому синие мазки корректно превращаются в экспертную маску.
- Добавлен устойчивый resolver путей в `app.py`: старые CSV с абсолютными Windows-путями больше не ломают вкладку экспертной проверки после переноса проекта.
- Новые записи в CSV/JSON сохраняют пути относительно папки проекта (`outputs/...`), а не абсолютные пути конкретного компьютера.
- Ручная разметка вынесена в браузерный JS-canvas: размер кисти, режим, undo и очистка работают без обращения к серверу.

## Исправления интерфейса в версии no-dim

- Добавлен жёсткий CSS-патч против затемнения Streamlit при rerun: stale-элементы больше не должны визуально приглушать canvas и основную область приложения.
- Ручная разметка стала легче: финальная маска и diff-overlay больше не пересчитываются автоматически при каждом изменении виджета.
- Добавлен выбор размера canvas:
  - `быстро` — до 650 px по длинной стороне;
  - `баланс` — до 850 px;
  - `детально` — до 1100 px.
- Тяжёлый предпросмотр diff по умолчанию выключен. Его можно включить отдельным чекбоксом.
- При сохранении экспертной оценки canvas считывается ещё раз, поэтому эксперт может не нажимать «Обновить предпросмотр» после последнего мазка.
- Добавлен `.streamlit/config.toml` с `fastReruns = true`, `runOnSave = false` и отключённой отправкой usage stats.
